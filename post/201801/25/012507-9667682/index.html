有哪些編程書，看完讓人大呼過癮？

有哪些你看了以後大呼過癮的編程書？

大學時我對編程語言有過一些困惑：應該學哪個編程語言？到底怎麼樣纔算是掌握了編程語言？5 年前我幸運地看到了垠神在新浪博客的一篇文章——如何掌握程序語言，裏面推薦閱讀 SICP 的前三章。我讀過之後，受益良多。

曾幾何時，SICP 是每個 MIT CS 學生大一就要上的基礎課的教材，畢竟是世界名校，比我們的課程不知高到哪裏去了。然鵝，時過境遷，MIT 的教授們已經停止這門基於 SICP 的課程了，因爲一方面他們已經教了太久（十幾年了），感覺很煩了，另一方面他們認爲軟件工程的學生現在把大部分時間花在閱讀軟件庫的說明書上，然後搞清楚怎麼調用，SICP 跟這套工作流程已經不怎麼相關了。

好了，雖說這門課已經不更新了，SICP 的作者承認他們並不知道應該如何安排更合理的課程，所以我們還是要看看這本書。

另外，基於這本書，伯克利的一位教授搞了個教學網站 - http://composingprograms.com/about.html，用 Python 來教 SICP，有興趣可以去看看。這個網站的內容也是基於伯克利的 CS 61A，這門課的名字就叫 SICP。實際上 MIT 也已經用 Python 替換了 Scheme，所以我個人更推薦上面這個網站，比起 Scheme，Python 更加適合初學者。不過 SICP 的內容不會過時，Scheme 只是一個表現形式，掃一遍前三章也絕對不虧。非初學者也會有時看時新的感覺。

SICP 幫助讀者瞭解 CS 的核心概念，屬於從高空俯瞰世界，剛入門的讀者不應該錯過。

說完了高空，接下來我想推薦另一本中間高度的書，這本書幫助我從瞭解如何組織代碼的模塊，以及實踐中業界遇到的問題，以及如何用合適的架構來解決這些問題。

Clean Architecture: A Craftsman's Guide to Software Structure and Design

這本書是 Uncle Bob 最近幾年的博文合集，讀完以後有種靈臺清明的感覺，做 code review 的時候思路清晰了很多。

看過差的架構才明白好的有多麼珍貴。架構都是一點點爛掉的，所以防微杜漸很重要，而要防微杜漸，就必須要從一開始就知道正確的架構方向。這本書不長，英文版也只需要不到 10 個小時就能讀完，大學裏或是工作中的讀者應該會喜歡這本書。這本書 2017 年 9 月纔出版，時效性遠遠超過經典的 Martin Fowler 的 Patterns of Enterprise Application Architecture，你如果準備讀後者，建議你先讀這本小書。

說完了中間高度，還有一本更加細節的書，我實在不能不推薦，因爲我看到別人沒提到，也許是因爲這本書 2017 年三月纔出版，還沒有中文的。

Martin Kleppmann 的 DDIA：Designing Data-Intensive Applications

這本書的作者閱讀量非常廣泛，以數據密集型軟件的設計爲楔子講了分佈式系統的很多東西，從設計思路到一些具體細節，收放自如。從 Netflix 的技術博客，到開源軟件的技術來源，再到 OSDI 上各個業界大廠發的論文，讓我大飽眼福。一共 12 章節，目前看到了第 9 章。

如果說上一本是講了高可擴展性、高可維護性的架構，是骨架；那麼這本書是講了分佈式、高併發、高性能、高可用的軟件，是血肉。細胞的級別講不出來，絕知此事要躬行。

微軟 CTO Kevin Scott 對這本書的評價是『每一位軟件工程師的必讀書。』

目前這本書在 Goodreads 上有 404 個評價，評分高達 4.68，要知道 SICP 才 4.45.

簡單強調一下：國內的朋友不要排斥閱讀英文資料，作者都是理工科，寫作詞彙量基本也就是四級英語，跟咱們高中英語差不多。
