互聯網行業哪個職位最有前途？來比作餐館試試……

互聯網行業哪個職位比較有前途？

珠玉在前，不過我還是想從另一個「形象」的角度來回答這個問題。

大部分互聯網公司的本質，是吸引更多的商戶（B）或用戶（C）來使用自己的產品（P），並使得他們願意爲其花錢買單。基於這個特性，我們可以把一家互聯網公司比作一家餐館，將互聯網公司的職位和餐館裏的職能一一對應起來。

對應關係大致是這樣的：

-

  • 研發工程師 - 準備食材；
  • 數據倉庫工程師 - 食材篩選、歸類 & 切菜；
  • 算法工程師 - 炒菜；
  • 運維工程師 - 洗碗 / 餐具歸類；
  • 產品設計師 / 產品經理 （PD/PM）- 設計菜單；
  • 產品運營 - 設計菜品的優惠活動和套餐等；
  • 數據分析師（BI）- 服務顧客；
  • 美術設計師 - 設計餐廳和菜品的形態；
  • 業務發展經理（BD）- 餐廳推廣；

（其他的生態部門，如 公關（PR）/ 客服 / 財務 / 人力資源（HR） 等，互聯網行業和傳統行業沒有太大差別，餐館裏也有對應的職位，這裏就不多講了。）

-

互聯網哪個職位最有前途，就好比餐廳裏哪個職能是最重要的一樣。

【一】

首先，食材準備（研發工程師）是不可或缺的。

“菜單都準備好了，就差食材了” 同樣也算是餐飲界的一個笑話。

獲取食材的方式可以是多種多樣的：可以去菜場買現成的（開發工程師），也可以自己種植（研究工程師），前者比較高效，但是選擇去哪個菜場買（使用語言）、如何控制原材料成本（代碼可讀性）和運輸成本（代碼複用性）都是值得探究的；後者比較低效，而且週期長，小餐館考慮到成本，一般不會這麼做，但是大餐館很願意親自試一試：畢竟如果發現了優秀的種植方法，可以形成壁壘，讓餐館具有很強的競爭力。

懂得食材準備的人，通常也會自己做菜，畢竟只有做過菜了，才知道不同食材的口感差別。因此對於小餐館來說，可能準備食材、篩選食材、切菜、燒菜 都是同一個人乾的。

【二】

專門僱一個人來篩選食材（數據倉庫工程師 - 底層），對於小餐館來說是很奢侈的。因爲小餐館賣的菜少，食材來源比較單一，數量也不多。但對於大餐館來說，由於食材的種類繁多，數量巨大，篩選食材也是一門技術活，不過時間長了，還是有點枯燥的。

篩選完食材後，就是歸類食材（數據倉庫工程師 - 中間層），對於小餐館來說，這一步同樣毫無意義，但是大餐館很講究這些，他們不僅會把食材按照 肉類 / 禽類 / 素菜 / 豆製品 / 菌菇類 等大類，對於相同類型的食材，他們也會按照食材的質量進行區分。他們這麼做，是爲了讓廚師儘快地挑選合適的食材進行加工，並且具備一定地應對菜單變化的能力。這份工作其實很有意思，因爲他能解決餐館廚房的效率問題。

食材歸類後，通常並不能直接吃，需要一個切菜的（數據倉庫工程師 - 應用層）。有些餐館比較簡單，專門賣涼菜，切好以後就可以直接給顧客了，有些則需要進一步加工。

【三】

對於大部分餐廳來說，炒菜（算法工程師）是必不可少的。頂級的廚師大多具有很出色地炒菜功夫，他們知道哪些菜放在一起炒比較好吃，每樣菜放多少調味品是合適的。

有時候，炒菜師傅覺得無聊了，會嘗試一些不尋常的搭配（數據挖掘），俗稱黑暗料理，有時候，黑暗料理也會令人意外地好吃呢。

炒菜的學問很多，最有意思的是，有些廚師會據歷史炒菜方式和顧客的反饋，來更新炒菜方式（機器學習），聽說這種能力最近非常吃香。

炒菜師傅通常只做標準化的菜餚，也很少直接對接顧客，但和切菜工及服務員的交流還是蠻多的。

【四】

餐廳的服務員（BI）是一個有點微妙的角色，他們需要和很多人交流，比如顧客、廚師、財務等，需要了解菜單上菜品的變化和各種優惠活動。

顧客是上帝，但上帝有時候不太講道理，會提出一些個性化的需求，比如：少放辣、不放香菜等，這使得他們不得不準備一個調味包（個性化報表）；有些比較奇葩的顧客，可能還會提出一些菜單上沒有的菜品。這時候，他們有三個選擇：

  • 告訴顧客沒有這個菜（拒絕需求），但顧客有時不是好惹的；
  • 自己去廚房看看有沒有相似的原料（臨時需求），讓廚師幫忙做，有時廚師幫不過來，還得自己親自做；
  • 通知制定菜單的人（PD），討論下有沒有必要修改菜單。

服務員可能是餐廳裏受氣最多的人之一，顧客罵他們，廚師也煩他們。

服務員未來的發展似乎不太明朗，這是因爲，很多餐廳已經推出了支付寶 / 微信掃碼自助點餐（聽說這是廚師搞出來的），這相當於替代了他們的其中一項核心職能。不過，作爲和顧客親密接觸的人，他們常常能接觸到第一手的評價，所以對顧客口味相當瞭解，也會做出一些有用的判斷和分析，他們一方面鄙視廚師不瞭解客戶需求，一方面卻又想親自下廚試試看。

【五】

毫無疑問，洗碗和處理餐具（運維工程師）是一項累活，但對於餐館尤其是大餐館來說，這又是必不可少的，因爲這決定了餐館的續航能力。

“沒有我們，餐廳的運作就會癱瘓。” 洗碗工們如是說。

【六】

設計菜單（PD）是一項表面上特別光鮮的工作，畢竟每個顧客都會看菜單，菜單的精美程度和菜品的豐盛程度決定着顧客的食慾。

但很多人不知道，菜單設計其實也是一項髒活，因爲他需要協調的人太多了，每次菜單的改動，他們需要去問準備食材的人是否能搞到原材料、問廚師是否擅長炒菜、還要說服服務員這個菜是值得加的（畢竟增加了服務員的工作量）。聽起來都是雜活。

但菜單設計又是至關重要的，他決定者顧客進入餐廳後，是第一時間離開，還是願意在這裏試吃一下。

【七】

“菜單決定了用戶的食慾，但我們纔是吸引用戶來的源泉。” 設計菜品促銷活動的小金（運營）如是說。

確實如此，如果餐館比較冷門，且菜品在價格上沒有優勢，連進門的人都沒幾個。

設置促銷活動，聽起來簡單，但其實也是一項技術活，因爲他們需要同時做到以下 2 點：

  1. 優惠力度儘可能小；
  2. 讓用戶覺得很划算。

看起來挺矛盾啊。可餐館不是慈善部門，花最少的錢獲取用戶纔好。

【八】

“如果沒有渠道，你們連用戶都接觸不到，怎麼吸引他們？”

負責餐廳推廣的小宸，一直感覺自己纔是最核心的角色。

其實他說的沒錯，對於一個剛剛成立的小餐館來說，廣發廣告，吸引到第一個用戶纔是最重要的，如果推廣方式得當，餐館的初創期就會顯著縮短。

【九】

“我可能不是核心人物，但是我會提升餐廳的逼格。”

負責菜品和餐廳的形態設計（美術設計），讓小黃感覺很有成就感。

看起來，色香味中，色似乎是最不重要的一項，畢竟菜是用來吃的。

“但好看的菜可以發到朋友圈啊！”

於是乎，小黃在不知不覺中決定了餐廳的文化，雖然這比起賺錢來說，它看起來很虛。

-

未來的餐廳會是怎樣的？

我想了解這個問題，於是採訪了餐廳的各個職能部門的員工：

  • “食材準備師” 小泰說：“我希望能找到菜品更全的菜場，更好的運輸工具，來降低食材的成本。”
  • “切菜工” 小白說：“我希望能造出一臺全自動切菜機。”
  • “炒菜師” 小風說：“我希望能多瞭解到顧客的口味，這樣就可以做出更貼合顧客口味的菜。”
  • “服務員” 小時說：“我希望能和顧客多聊聊天，比如算命什麼的，這樣我可以讓顧客因爲我貼切的服務而選擇留下。”
  • “促銷活動設計師” 小金說：“我希望能多學點數學和經濟學，這樣才知道怎麼高促銷才合適。”
  • “餐廳和菜品設計師” 小黃沉迷在自己的設計中，沒興趣搭理我們。
  • “餐廳推廣經理” 小宸正在出差，電話忙音。
  • “洗碗工” 小季則去旅遊了，畢竟全自動洗碗機很好用，大部分時間不用他們出場。
  • 至於“菜單設計師” 小軒，我們一直沒有找到他，因爲他等概率地出現在餐廳的各個位置，被恰好碰到是一個小概率事件啦。
