在接到一次 Google 的烏龍電話之後，我想，這真是家驕傲的公司

谷歌的驕傲，驕傲的谷歌

昨天發生了一件事情。電話突然響起來，接電話一聊發現是消失已久的谷歌的 recruiter（招聘職員）。對方問我爲什麼發了好幾次郵件我卻一直不回。這讓我大吃一驚。因爲我從未接到過郵件，而且這是無法想象的奇蹟。

我在 2009 年到 2011 年期間經由不同朋友推薦鼓動，毫無刷題準備的情況下面試谷歌三次。三次 onsite（現場考覈）都掛掉以後，再也沒有谷歌 recruiter 找我。這個困擾我的事情在 Quora 上有一個答案。一個資深 recruiter 透露谷歌內部有規定，任何人一輩子只有三次 onsite 面試谷歌的機會，三跪谷歌，谷歌就會永久凍結這個人了。

傳聞看起來是真的，所以谷歌 recruiter 再來找我是個奇蹟。我覺得奇蹟不應該發生在我身上。於是我問他：你確信找的人是你要找的，而不是同名同姓的另外一個人嗎？

對方問我爲什麼。我說聽聞谷歌有這個三跪就永久凍結的政策，我在 2009 年到 2011 年跪了三次，這輩子你們應該不會來找我了。不知道出於什麼原因，對方表示這個政策不但是真的，而且到現在依然有效。

後面的故事就是對方發現果然是另外一個和我名字一樣的人，被填上了我的 Linkedin 的主頁。於是有了這個烏龍。Recruiter 說不好意思打攪你，但是你還是永久被禁，沒資格面試我們的。

這個機會難得，我就多和 recruiter 聊了幾句。不但證實了谷歌確實一個人一輩子只有三次機會。而且 recruiter 告訴我，符合兩種情況之一的可以申請開 exception，如果 exception 被批准了，我就可以獲得額外的一次面試機會。第一個是我有一個新的更高的學位。當然這個沒戲了，我已經 PhD 畢業了。第二個是谷歌內部的大牛力推我。後者估計也沒什麼戲了。我更不知道的是如果這個面試再掛掉的話，是不是一輩子連 exception 的資格都沒有了。

三跪谷歌不是什麼光彩的事情。但是這個事情很多人也都知道，所以也不是什麼不可以公開的事情。只能說我既不是天才準備也不充分。不過這個政策的確是非常的值得我們探究一下。

谷歌面試有很大的隨機性，這個很多人都知道。進了谷歌的人，如果拉出來重新走一遍面試流程，很多人估計也會被拒掉。這個背後的原因很多人也討論過，基本上來說，這個流程裏面招到的人，肯定都是人才。至於那些因爲這個過程被隨機出去的人才，那就只能說抱歉，命不好怪上帝吧。

我想主要的原因還是谷歌有錢，又看起來很高大上，申請的人很多，所以它可以任性，想怎麼幹都行。谷歌是驕傲的，方方面面都可以體現出這種驕傲來。

我記得 2008 年的時候，谷歌三駕馬車已經很成熟。Hadoop 就像是一個玩具一般。當時谷歌和 IBM 聯合組團包了一個廢棄的數據中心。在數據中心裏面裝上 Hadoop，供全球的研究人員使用。谷歌覺得展現 MapReduce 的偉大可以通過這種舉動來實現。當大家發現 Hadoop 這個不怎麼樣的軟件都能夠做很多事情的時候，谷歌內部的基礎架構的偉大自然而然也體現出來了。

與此同時，谷歌很早進入到雲計算了。比亞馬遜當然晚一點，但是比微軟早多了。只不過谷歌眼裏主推的產品是 AppEngine，不是賣虛擬機。至於類似亞馬遜的虛擬機服務，那就得等到 4 年以後才以 Compute Engine 的名義推出。箇中緣由，最重要的還是谷歌覺得它的路線就是最好的，全世界只需要 follow 就好。

AppEngine 當然不全是沒有意義的東西，有很多值得稱道的地方。但是裏面主推的是 BigTable 這樣的 Key-Value Store。我在 2008 年硅谷某個活動場合裏見到 AppEngine 的人在宣傳。有人在下面問，數據存儲連 Join 都做不了，怎麼用？谷歌的人回答說，我們在內部從來都不用 Join 的，BigTable 是未來。弄得下面的人一臉懵逼的不知道怎麼接。谷歌宣傳 Join 已經過時，不是必需品，並非只有那次講座。

當然後來事情的發展大家都知道了，谷歌做了個 Spanner 的東西。這個東西很牛非常牛。但是單純從數據模型看，這多少是自食其言了。

Spanner 成了谷歌雲上的服務。先進的技術，應該是皆大歡喜的事情。但是 Spanner 的那個 SQL 有點糟心。沒有人知道這個 SQL 到底兼容 SQL 的哪個版本。至於爲什麼會這樣，和谷歌內部的人聊天就知道緣由了。谷歌的人覺得 SQL 標準在技術上並非最合理，而谷歌自己做的決定更合理。至於客戶是不是願意爲了谷歌更先進的技術改寫自己的程序，那就不是谷歌的人認真想的問題了。

一方面我覺得谷歌的命其實挺好的。谷歌的搜索引擎和 Page Rank 算法當然是獨步天下的。但是谷歌現在的印鈔機，通過競價排名來賣廣告的基本思想並非原創，抄襲了 Overture。當然我們不可否認，谷歌技術特別牛，抄東西抄的不錯。

Overture1998 年就開始做這個生意了，谷歌進入新世紀纔開始。Overture 和谷歌的官司打了好久。只不過在雅虎收購了 Overture 以後，以區區 250 萬股谷歌原始股就和解了。更重要的，這些股票還被雅虎當時的首席財務官以 82 美元一股在谷歌上市前賣給了投資財團。只能說，雅虎這個傻公司，不滅亡真沒天理。

倘若那場官司最後以每筆交易收多少專利費的形式結束的話，誰知道今天怎麼樣呢？但是不管怎麼樣，谷歌賺錢靠廣告，其他方面努力了這麼多年，依舊沒看到多少錢冒出來啊。

本來吧，大數據這一塊谷歌有很大概率主導的，但是最後成了 Hadoop 的天下，讓亞馬遜賺足了錢。而谷歌自己的 BigTable 做雲計算服務，提供 API 的時候，居然需要提供和 HBase 兼容的 API。這也是極少數的谷歌爲了利益而屈服於所謂落後的技術，保證兼容性的舉措了。但是這個舉措的背後所反映的，其實還是那個驕傲的谷歌和谷歌的驕傲帶來的問題。

我一直覺得谷歌這麼人才濟濟的公司，本來可以做得更好的。奈何谷歌覺得自己就是牛逼啊，世界都要圍着谷歌轉纔是真理。谷歌爲了這個世界而妥協，實在是不符合其一貫的作風啊。

所以最後往往就是谷歌有最牛的技術，賺錢的卻是亞馬遜。雖然我對亞馬遜這個公司某些方面觀感很一般，但是亞馬遜的 leadership Principal 裏面有一些，是非常接地氣的東西。如果谷歌有同樣接地氣的能力，谷歌何止是今天的谷歌呢？

當然，其實沒辦法，每個公司都有自己的宿命。一羣很聰明又覺得自己纔是真理的人領導的公司，我想可能也就是這樣的公司了。一方面谷歌很偉大，做了很多很厲害的技術。另外一方面谷歌真的是非常的驕傲。讓我不知道要怎麼說。

我做谷歌黑有一段日子， 所以我的觀點當然是有偏見的。不過我起碼也是有理有據的表達和證明我的觀點。我不太喜歡上來就罵孃的留言，但我歡迎有理有據的觀點，無論同意還是反對我的。
