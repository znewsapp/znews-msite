畢業後進了外包公司，本想着都是寫代碼，應該沒什麼差別......

爲什麼同樣是程序員感覺外包和互聯網之間隔着一個不可逾越的鴻溝？

1.人員來源。技術上第一梯隊的畢業生進入了一線頂級互聯網公司，國企，知名外企，以及部分優秀的高福利創業公司。之後以此類推，分別是二三線互聯網公司，初創公司。

最後纔是外包公司，外包又分軟件外包和人員外派。人員外派技術要求相對較嚴格一點，因爲有甲方公司的二次面試。軟件外包的門檻就基本沒有了，會寫代碼就成。

我們不排除有個別優秀的人不在五行之中，因爲各種原因沒有按照這個梯隊對號入座，但大方向是這樣的，這事關薪資成長等福利待遇。

2.開發環境。

互聯網公司的開發模式，只要項目不死，那就一直保持着持續迭代的過程。這期間隨着市場變化，開發人員將會不停的接受新的挑戰，需要學習新的知識以應對。而隨着公司體量的越來越大，用戶越來越多，需求越來越多，對軟件質量，開發技術的挑戰也越來多，逼迫開發者需要對技術進一步深入研究以應對！

軟件外包則要簡單的多了，大部分都是一錘子買賣，在幾個月到半年一年的時間裏開發出一套可用的系統，這裏面幾乎是不需要考慮後期迭代和維護的，外包公司的核心目的只有一個快！做完這單接着下一單。我們不說 bug，只說代碼質量，就知道有多差。你可能會說了，這還是看人的吧，我就願意把代碼寫好看寫標準了，嗯…隨着高強度的需求量我不知道你能堅持多久。軟件外包幾乎是不需要比較深的技術探索的，保持在可用級別就好。

人員外派相對軟件外包要好一點，畢竟甲方自己的項目不是一錘子買賣，人家還要維護迭代好久，但其實大部分時候甲方是不會把核心功能交給外派人員來做的，基本都是一些簡單的體力活…所以技術成長度可想而知。

總結一下，因爲行業中的人員來源決定了外包行業不會有太多優秀的開發人才，同時又因爲行業特性決定了外包行業的人不需要過於深入的技術探索，即使個別同學會主動學習，但高強度的普通需求壓力下，這種學習的阻力真的很大。

當然也不是說互聯網公司就一定都在研究高大上的技術，但整體環境要比外包高出太多。

初始人員來源加上後期成長的環境，導致兩個行業的差距越來越大，鴻溝也就出現了。

所以建議樓主，趕緊跳出來吧…

----------

金融股票公司的軟件我沒了解過就不談了，但長期維護的外包，某種程度上這種外包已經完全區別開大部分一錘子買賣的外包了。

代碼質量能當飯吃？嘿嘿，還真能當飯吃。能用就行了 出過啥問題？

這年頭還有人敢說自己寫的代碼絕對沒 bug？？爛代碼等出問題的時候你就忙的吃不上飯了，各種奇葩邏輯，命名，找原因找半天，好不容易找到原因要改了，發現耦合度太高，牽一髮動全身，要不然一改就要改好多地方，要不然就是改出其他一堆 bug。 這只是代碼質量層面的問題，要是架構設計上出現問題，分分鐘想要推到重寫也不爲過。

一個是出問題，另一個是加需求。爛代碼的擴展性可想而知，在這上面加需求不如重寫。

一錘子買賣的外包完全不需要考慮後期擴展，一味追求速度，最終實現也僅追求可用級別，性能什麼的就更不用提了。

長此以往養成的開發習慣，大的互聯網公司是無法接受的！

我來心平氣和的說幾句話。

我現在是一個外包的公司的負責人，外包公司是當時創業項目失敗了之後，爲了活着找到的勉強方法。

這三年我試了幾次次創業型項目，後面都是和外包公司並行。經濟公司，打車比價，分答或模板銷售，遊戲工作室。

我都放棄了。原因有好些個。

外包不是個受人尊敬的活，我和我的客戶說，也和我的小夥伴說，也和我們合夥人說，但爲了生存我不能停。

我現在公司 46 個人，三十來個開發。我一個全職業務，有兩個小夥伴幫我帶着跑。主要壓力還是要自己扛。

我目前總共拿過四家公司的股份，我們的客戶。有技術投資，有拿了成本，有我幫他們搞運營和流量。有 A，有天使。

一家股東撕逼，我直接沒法參與。 私教健身。

一家運營不算好，雖然 A，但我平價退出。 設計。

一家方向有些問題，現在調整。 深度旅遊。

還有家經營不善，不知道能活多久。美甲 O2O+ 培訓。

我現在有六個微信，一萬八千左右好友。加上我給小夥伴的號，兩萬三四。

我一週有四五天在各地跑，上海一天高峯四五個地方，一般兩三個。經常上海繞圈。

這是我，我還有個技術合夥人，性格溫和，技術大牛，熟悉信息化系統。

現在我的 Team Leader 是 14 年跟着我們創業的那幾個技術。

我們現在做了大概七十多個項目。

外包有什麼問題：

1 項目無法深度參與，很多甲方很獨當和強勢，沒有客觀的接受我們的意見，更多時候就是你說我做，最後難免敷衍。當然很多時候有解決辦法，真正當成自己的產品打磨和思考，但客戶一旦不接受這裏面的價格，那麼就很難。我們有些大客戶，光原型就做了考慮和碰撞，他願意買單，我才花得起時間。但大部分創業項目都窮，大部分企業項目利潤都被中間人拿走。

2 養大的孩子和別人跑，全部在餵奶過程。前期是最需要磨合和適應的過程。項目做好了，客戶就會融資自己接過去，看着好的沒自己的份，沒有一個心裏伴隨感和技術的成長性，對自我技術的發展不好。今天我們 JAVA 負責人就和我說老是做這個項目對他的職業路勁不好，正好我們有個百萬級用戶的客戶，接下來他抗。項目開發好了，但做砸了，只能默默罵運營傻逼。無能爲力。很多時候還拿不到錢，就可能發不出工資。一旦項目出了問題，客戶就理直氣壯我項目不做了，他知道你對，他知道該給你錢，他知道項目成功失敗和你沒關，就是不給你錢。

3 項目小而砸，沒法專心做一個事情。我們有些客戶公司體量上去了，每個人的職業路徑是清晰的，那麼他更多是在這條路上去走和摸索，就會不錯。但外包公司可能同時要做幾個項目，耗腦力還要耗心力。程序員大都反感做一個拼湊沒有延續性的事情。

爲了解決這些問題。

我努力的在試。

剛剛談妥拿那個一百多萬用戶項目三個點股份。我們技術核心成員會有份額，我代持。

我們參與了兩個區塊鏈項目，其中一個發幣。我也拿了一些份額，一半公司，一半參與人員平分。

他們對於目前未可視化的東西其實渴望不大，當然他要不要，和我給不給不是一個維度的事情。

所以我該做的還是做。

我們核心的開發都有公司的股份，不多，整體發出去 15%。這三年半有人要投我們，信我這個人。也有大公司要收，沒賣。

我們目前發的工資在市場是持平的，沒有你說低於，現在還在爲這一次漲薪愁，先漲哪一批。

明確了績效考覈。

我希望做到的是：外包養活大家，有明確的計算規則，讓多幹的人多拿，讓穩定的人持平，讓老油條離開。

幣上面也是，我自己先掏錢去測試靠譜的，然後讓他們認購。

我們公司從來不是死氣沉沉的。

JAVA 把我刪過好友，嫌我煩。

產品經理護犢子把讓我閉嘴。

吐槽就不復述了。

我不在意這些，只在意客戶給我的東西能不能做好，很多時候這真的是信我這個人才給我這個項目。

但有些基本的規則必須遵守，像人的基本尊嚴和商業合約。

我被威脅拿到正常合同前期款放棄增訂爲了發工資我只能妥協，也被嚇唬說你不知道他社會上是幹啥的只因爲我催着要我應得的開發款；我在停車場等了幾小時上去被很敷衍的打發走，也被很多幫助理解成理所當然。

這個社會其實不那麼友好。

我也覺得對不起這些小夥伴，所以在我儘可能的範圍內去嘗試，尋求一些變數和可能性，我不知道公司會不會被我這樣的嘗試搞死，但我如果默認這樣的平庸而心安理得會被自己羞愧死。

我希望能外包給大家發不低於市場的工資，然後他們比外面人更多努力一點和相信我們一點，儘可能的參與和佔有一些好的行業和項目的額度。教育，區塊鏈，醫美，深度旅遊。

不斷的給他們拉皮條，投資，上下游資源，渠道，推廣。

希望他們成，那我就可以給小夥伴一些想象的可能性和財富的回報率。

我創業不是爲了外包，但外包沒有錯，只是我做的不是最好。

只是再給我一點點時間。

謝謝。
